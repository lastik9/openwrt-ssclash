# openwrt-ssclash

**Русский** · [English](https://github.com/lastik9/openwrt-ssclash/blob/main/README.en.md) · [中文](https://github.com/lastik9/openwrt-ssclash/blob/main/README.zh-CN.md)

Установочный скрипт для **SSClash** — прозрачного обхода блокировок на **OpenWrt 25 (apk)** через ядро **Mihomo (Clash.Meta)** и веб-панель [luci-app-ssclash](https://github.com/zerolabnet/SSClash).

Один прогон на чистой системе: ставит зависимости и последнюю версию панели, а ядро Mihomo либо качает сразу из консоли (по подтверждению), либо оставляет на кнопку в LuCI. Есть отдельный аптинсталлер для чистого удаления.

## Зачем

[luci-app-ssclash](https://github.com/zerolabnet/SSClash) — обёртка над ядром [Mihomo](https://github.com/MetaCubeX/mihomo), которая заворачивает трафик роутера через прокси (VLESS/Shadowsocks/Trojan и т.д.) по правилам из твоего конфига. Ставится это в несколько ручных шагов с фиксированными версиями пакета и ядра. Скрипт снимает рутину: сам берёт **последние** версии панели и ядра с GitHub и сам определяет архитектуру роутера под нужную сборку Mihomo.

## Требования

- OpenWrt 25.x с пакетным менеджером **apk** (для сборок на opkg скрипт не предназначен).
- Доступ в интернет на роутере на момент запуска — качаются пакеты, панель и ядро с GitHub.
- Немного места в `/opt` под ядро Mihomo (~15–30 МБ в распакованном виде).

## Что делает скрипт

1. `apk update`, установка зависимостей (`curl`, `kmod-nft-tproxy`, `kmod-tun`, `coreutils-base64`) и загрузка модулей ядра `tun`, `nft_tproxy` — чтобы перезагрузка не требовалась.
2. Определяет последнюю версию **luci-app-ssclash** через GitHub API и ставит её (`apk add --allow-untrusted`).
3. *(Опционально, с подтверждением)* определяет арку роутера через `apk --print-arch`, находит под неё последнее ядро **Mihomo** и распаковывает его в `/opt/clash/bin/clash`.
4. Оставляет сервис **остановленным** и **не перезагружает** роутер — чтобы ты сначала зашёл в панель, залил конфиг, а потом запустил Clash сам. Автозапуск при загрузке скрипт **не трогает** (пакет включает его по умолчанию).

Если ядро не качать в п.3 — его в один клик поставит сама панель: **Services → SSClash → Settings → Mihomo Kernel Management → Download Latest Kernel** (она тоже определяет арку сама).

## Установка

Команды выполняются **на роутере** (по SSH), а не на компьютере:

```
wget https://raw.githubusercontent.com/lastik9/openwrt-ssclash/main/setup-ssclash.sh
sh setup-ssclash.sh
```

Запуск покажет меню:

```
  1) Полная установка: панель + ядро + проксирование (спросит)
  2) Только панель + ядро (установка/обновление)
  3) Доустановить проксирование «за белыми списками»
  4) Убрать проксирование
  0) Выход
```

Для первой установки выбирай **1**. Пункты **3/4** удобны потом: если поставил без проксирования, его можно доустановить (или убрать) отдельно, не переустанавливая всё. Меню требует запуска файлом (`sh setup-ssclash.sh`), через `| sh` оно работать не будет — вместо него можно указать действие аргументом: `install`, `app`, `bypass`, `unbypass`.

После установки Clash **остановлен**. Открой в LuCI **Services → SSClash**, залей свой конфиг Clash/Mihomo и запусти сервис — кнопкой в панели или командой:

```
/etc/init.d/clash start
```

Автозапуск при загрузке уже включён по умолчанию, так что после перезагрузки Clash поднимется сам.

Обычно арку определять не нужно — скрипт делает это сам. Приставка `ARCH=` требуется, только если автоопределение ошиблось. Задаётся она **при запуске**, в той же строке (это переменная окружения, а не отдельный шаг):

```
sh setup-ssclash.sh             # обычный запуск — арка определяется сама
ARCH=arm64 sh setup-ssclash.sh  # запуск с принудительной аркой
```

Возможные значения `ARCH`: `amd64-compatible`, `amd64`, `amd64-v3`, `386`, `arm64`, `armv7`, `armv6`, `armv5`, `mipsle-softfloat`, `mips-softfloat`, `mips64le`, `mips64`, `riscv64`, `loong64`.

## Если GitHub режут (актуально для РФ)

С мая 2026 доступность GitHub из РФ нестабильна — бывают выборочные обрывы при скачивании. Скрипт уже сам повторяет попытки (`curl --retry`), поэтому часто достаточно просто запустить ещё раз. Если не помогает — два способа обойти.

Через свой заграничный сервер (в скрипте менять ничего не нужно — `curl` и `apk` уважают `https_proxy`):

```
https_proxy=http://ЗАГРАН:PORT sh setup-ssclash.sh app
```

Через зеркало GitHub — переменная `MIRROR` оборачивает и API, и скачивание релизов:

```
MIRROR=https://gh-proxy.com/ sh setup-ssclash.sh app
```

Подставь публичный GitHub-прокси (пример выше — проверь, что он живой) или адрес своего reverse-proxy; свой надёжнее. Пусто (по умолчанию) — прямой GitHub, в регионах без блокировок делать ничего не нужно.

## Обновление «за белыми списками»

Если провайдер режет доступ к репозиториям OpenWrt/GitHub напрямую (например, мобильный интернет в режиме «белых списков»), обновлять пакеты роутера можно через прокси Mihomo, который у тебя уже работает: трафик самого роутера (`apk`, `curl`) пускается на локальный прокси `127.0.0.1:7890`.

Настраивается в два места.

**1. Конфиг Mihomo** (заливается через LuCI — правишь сам). Открой локальный порт и добавь правило, отправляющее домены обновлений через прокси-узел:

```yaml
mixed-port: 7890

rules:
  - DOMAIN-SUFFIX,openwrt.org,PROXY
  # при необходимости — github.com, githubusercontent.com и т.п.
```

Порт желательно слушать только на localhost (`bind-address: 127.0.0.1`), чтобы не превращать роутер в открытый прокси в LAN.

**2. Переменные окружения роутера.** Это делает `setup-ssclash.sh` — при установке спросит «Настроить проксирование трафика роутера?» (пункт **1** меню). Доустановить или убрать это можно и потом — пунктами **3/4** меню (или `sh setup-ssclash.sh bypass` / `unbypass`), не переустанавливая всё. По согласию создаётся `/etc/profile.d/ssclash-proxy.sh`:

```sh
if pidof clash >/dev/null 2>&1; then
  export http_proxy="http://127.0.0.1:7890"
  export https_proxy="http://127.0.0.1:7890"
  # LAN-адрес роутера берётся на лету через uci — без привязки к конкретному IP
  _lan="$(uci -q get network.lan.ipaddr 2>/dev/null)"
  export no_proxy="127.0.0.1,localhost,::1${_lan:+,$_lan}"
  unset _lan
fi
```

Проверка `pidof clash` важна: прокси включается, только когда ядро запущено. Если Mihomo не работает (упал или ещё грузится после перезагрузки), переменные не выставляются — и `apk`/`curl` не ломаются, пытаясь достучаться до мёртвого порта. LAN-адрес роутера в `no_proxy` не захардкожен, а определяется при каждом входе через `uci`, поэтому файл одинаково работает на любом роутере. Настройка действует в новых SSH-сессиях (профиль читается при входе).

Разово, без правки профиля, то же самое можно проверить прямо в текущей сессии:

```
export http_proxy=http://127.0.0.1:7890
export https_proxy=http://127.0.0.1:7890
apk update
```

Файл `/etc/profile.d/ssclash-proxy.sh` удаляется аптинсталлером автоматически.

## Удаление

Удалить можно двумя способами — оба вызывают одну и ту же логику:

- пунктом **5** в меню `setup-ssclash.sh`;
- отдельным аптинсталлером (на роутере):

```
wget https://raw.githubusercontent.com/lastik9/openwrt-ssclash/main/uninstall-ssclash.sh
sh uninstall-ssclash.sh
```

Сначала спросит подтверждение, всегда удаляет сам SSClash, а в конце **отдельно спрашивает**, сносить ли ещё и общесистемные зависимости. Разница такая:

**Обычное удаление** (выполняется всегда) убирает то, что относится именно к SSClash: пакет `luci-app-ssclash`, весь каталог `/opt/clash` (ядро Mihomo и твои конфиги) и кэш меню LuCI. После этого SSClash на роутере не остаётся — в 99% случаев этого достаточно.

**Полное удаление** (если ответить «да» на вопрос про зависимости) вдобавок удаляет системные пакеты, которые SSClash тянул для работы:

- `kmod-tun` — модуль ядра для TUN-интерфейсов;
- `kmod-nft-tproxy` — модуль nftables для прозрачного проксирования;
- `coreutils-base64` — утилита base64.

Загвоздка в том, что это **общие** пакеты: их могут использовать и другие вещи на роутере (VPN вроде OpenVPN/WireGuard-обвязок, другие прокси, разные скрипты). Если удалить их, а что-то ещё на них опиралось — то что-то сломается. Поэтому по умолчанию скрипт их **оставляет**: места они занимают копейки и никому не мешают. Соглашайся на их удаление, только если роутер выделен под SSClash и ты уверен, что больше их ничего не использует. `curl` не удаляется в любом случае — он нужен слишком многим.

## Соответствие архитектур

| OpenWrt (`apk --print-arch`)   | Сборка Mihomo         |
| ------------------------------ | --------------------- |
| `aarch64_*`                    | `arm64`               |
| `x86_64`                       | `amd64-compatible`    |
| `i386_*`                       | `386`                 |
| `arm_cortex-a7/a9/a15…`        | `armv7`               |
| `arm_arm1176*` (Pi1/Zero)      | `armv6`               |
| `arm_mpcore/fa526/xscale…`     | `armv5`               |
| `mipsel_24kc`                  | `mipsle-softfloat`    |
| `mips_24kc`                    | `mips-softfloat`      |
| `mips64el_*` / `mips64_*`      | `mips64le` / `mips64` |
| `riscv64_*`                    | `riscv64`             |
| `loongarch64_*`                | `loong64`             |
